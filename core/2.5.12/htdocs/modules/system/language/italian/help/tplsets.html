Guida: Temi

Descrizione

XOOPS consente una certa separazione tra la presentazione effettiva del sito (la “skin” o interfaccia utente) e il codice che gira nel back-end. Il primo è gestito da un meccanismo sofisticato basato su temi grafici. Un tema può includere, nella sua forma base, i template HTML essenziali che definiscono l'interfaccia visiva del sito, nonché i fogli di stile e le immagini necessarie per completare quella interfaccia visiva. I temi sono selezionabili nell'interfaccia amministrativa, quindi cambiare l'aspetto dell'intero sito XOOPS è semplicemente una questione di modificare il valore di un selettore e fare clic su "Accetta". XOOPS fornisce anche agli amministratori del sito la possibilità di permettere agli utenti di scegliere tra alcuni temi disponibili.

In generale, un tema è la struttura visiva di alto livello di un sito: il tema definisce, ad esempio, il layout generale del sito; d'altra parte, solitamente non definisce la distribuzione specifica dei contenuti all'interno di un blocco. Questo è gestito da un'entità diversa chiamata set di template. Un set di template è un insieme di template HTML che definiscono la presentazione dei contenuti all'interno di tutti i blocchi che formano un sito XOOPS. Se vogliamo una metafora più chiara per questo, potremmo dire che il tema è come l'architetto di una casa, mentre il set di template è come l'arredatore d'interni. Il primo definisce le pareti e la distribuzione generale, ma è il secondo che si occupa di “vestire” quelle pareti.

Se il tema attivo è un tema semplice che include solo un file di tema, file CSS e alcune immagini, XOOPS impiega il set di template predefinito per definire la presentazione dei blocchi. Tuttavia, alcuni temi complessi sono abbastanza sofisticati da includere il proprio set di template. In questo caso, l'amministratore del sito deve utilizzare l'interfaccia amministrativa per impostare sia il tema che il set di template.

Come viene realizzata la magia del tema? Cioè, come possono i template HTML includere contenuti dinamici? Questo non viene fatto con specchi e fumo, ma con un astuto motore di template chiamato Smarty che svolge la funzione di quello che un tempo veniva chiamato interpolazione di variabili. Esprimendolo in modo semplice, potremmo dire che il sistema XOOPS usa file PHP per contenere la logica dei moduli (accesso al database, allocazione di variabili, gestione dei diritti...), file HTML con variabili Smarty inserite per gestire l'interfaccia visiva effettiva, e il motore Smarty per inserire valori e stringhe in posizioni specifiche all'interno dell'HTML che costruisce le pagine viste dall'utente finale. In sintesi, quello che questo significa per te può essere facilmente riassunto: grazie al suo sistema di temi, XOOPS consente ai designer di siti di creare siti più interessanti e visivamente sorprendenti senza doversi preoccupare di ciò che accade sotto il cofano.

Clonare e scaricare set di template

Se desideri creare un set di template personalizzato, puoi fare una copia del set predefinito (o di qualsiasi altro) selezionando il link ‘clona’ (ti verrà chiesto di dare un nome al nuovo set). A differenza del set predefinito, altri set di template possono essere modificati liberamente. Puoi anche salvare una copia di un set di template sul disco selezionando il link ‘scarica’, o rimuoverlo con il link ‘elimina’. Potresti anche vedere occasionalmente un link aggiuntivo ‘Genera’ accanto a nuovi moduli per i quali i template non sono ancora stati creati.

Modificare i template

Puoi vedere un elenco completo dei template disponibili navigando nell'albero delle directory. Quando selezioni un template, XOOPS lo aprirà in un editor dove puoi modificarlo.

Durante la modifica dei template, devi abilitare una preferenza che aggiorna automaticamente i file .tpl del modulo se ci sono file più recenti nella directory themes/iltuoTema/templates per il tema corrente (Modulo di Sistema -> Preferenze -> Impostazioni generali -> Aggiornare i file template .tpl del modulo dalla directory themes/iltuoTema/templates? -> ‘Sì’). Questa opzione dovrebbe essere disabilitata una volta che il sito è online.

Se accidentalmente modifichi un template, non dimenticare che puoi recuperare una copia ‘buona’ dal set di template predefinito. Se stai lavorando con template non standard, potrebbe essere una buona idea salvare una copia su disco prima di iniziare a modificarli.
